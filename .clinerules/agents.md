# La Cucina - Правила для ИИ-агентов

## Архитектура и Стек
- Проект: Монорепозиторий
- Фреймворк: Next.js 15+ (App Router)
- API Слой: tRPC. Все мутации и запросы только через tRPC роутеры в server/api/routers
- БД: PostgreSQL + Drizzle ORM. Схема в server/db/schema.ts
- Аутентификация: Better Auth. Роли: user, admin, waiter
- Стейт: Zustand для корзины, React Query для серверного стейта
- Стили: Tailwind CSS (тёмная тема)

## Стиль кодирования
- Язык: TypeScript. Строгая типизация. Никакого any
- Именование: camelCase для переменных, PascalCase для компонентов
- Импорты: Использовать алиас @/
- Форматирование: Одинарные кавычки, запятые в конце строк

## Запрещено
- НЕ создавать REST API Route (app/api/...), кроме tRPC и Better Auth
- НЕ использовать fetch на клиенте для запросов к своему серверу
- НЕ игнорировать типы
- НЕ хардкодить цвета — использовать классы Tailwind
